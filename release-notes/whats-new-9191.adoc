| link:../learn-more/additional-supported-ontap-features.html#san-direct-attachment["Supporto per configurazioni FC ad aggancio diretto"] | Non è necessario uno switch FC per stabilire connessioni tra host FC e FC-NVMe e storage ONTAP. [[security]]
== Sicurezza

[cols="2,4"]
|===
| Aggiornare | Descrizione 


| link:../secure-data/enable-anti-ransomware-protection.html["Supporto per ARP/AI con SnapMirror relazioni di sincronizzazione attive"] | La protezione autonoma contro i ransomware con intelligenza artificiale (ARP/AI) supporta le unità di storage primario di lettura/scrittura in relazioni di sincronizzazione attiva SnapMirror sui sistemi ASA r2. ARP/AI viene eseguito sulle unità di storage primario attive, crea snapshot ARP solo primarie e fornisce il rilevamento dei ransomware basato sull'apprendimento automatico per tali unità di storage. 


| link:../learn-more/additional-supported-ontap-features.html#tls-hardware-offload["Supporto per l'offload hardware TLS"] | È possibile delegare la crittografia e la decrittografia Transport Layer Security (TLS) dell'intero cluster alle schede Ethernet supportate. Ciò riduce il carico sulla CPU e migliora le prestazioni dei sistemi che utilizzano TLS. 
|===


[[storage-efficiency]]
== Efficienza dello storage

[cols="2,4"]
|===
| Aggiornare | Descrizione 


| link:../learn-more/additional-supported-ontap-features.html#global-deduplication["Supporto per la deduplicazione globale"] | La deduplicazione globale estende il risparmio di spazio derivante dalla deduplicazione ONTAP all'intera Storage Availability Zone (SAZ). Eliminando i blocchi di dati duplicati in tutte le unità di storage della SAZ, è possibile migliorare significativamente l'efficienza dello storage e ridurre il costo totale di proprietà. La deduplicazione globale è abilitata per impostazione predefinita e opera in modo trasparente senza necessità di configurazione amministrativa. 
|===
